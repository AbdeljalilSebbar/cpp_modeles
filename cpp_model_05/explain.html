CPP Module 05 - Bureaucracy & Exceptions

Deep dive into Exceptions, Class hierarchy, and proper error handling in C++

Overview

This module focuses on understanding exceptions and applying OOP principles such as encapsulation, class design, and error control. You’ll simulate a bureaucratic system where forms must be signed and executed under strict grade conditions.

Key Concepts

  • Exceptions: Handling runtime errors using try / catch / throw.
  • Custom Exception Classes: Creating user-defined exceptions inherited from std::exception.
  • Nested Classes: Defining exception classes inside main classes.
  • Const members: Declaring immutable data like a bureaucrat’s name.
  • Encapsulation: Protecting class data and using getters for controlled access.

Exercise 00: Bureaucrat

You create a Bureaucrat class with:

  • const std::string name
  • int grade (1 = highest, 150 = lowest)

It throws exceptions if the grade is out of range. You must handle:

  • GradeTooHighException
  • GradeTooLowException
try {
  Bureaucrat a("Alice", 151);
} catch (std::exception &e) {
  std::cout << e.what() << std::endl;
}

Exercise 01: Form

Add a new class Form that can be signed only if the bureaucrat’s grade is high enough. It has:

  • const std::string name
  • bool isSigned
  • int gradeToSign
  • int gradeToExecute

The Bureaucrat can attempt to sign it using:

void Bureaucrat::signForm(Form &form);

Inside the Form class:

void beSigned(Bureaucrat const &b);
// Throws GradeTooLowException if b.grade is too low

Exercise 02: AForm (Abstract Class)

Now you introduce inheritance and abstraction. Form becomes AForm — an abstract class with a pure virtual method:

virtual void execute(Bureaucrat const &executor) const = 0;

This means each specific form will define how to execute itself.

Exercise 03: Specific Forms

You implement three derived classes:

  • ShrubberyCreationForm: Creates a tree ASCII art file.
  • RobotomyRequestForm: 50% chance of success.
  • PresidentialPardonForm: Prints a pardon message.

Each overrides execute() and checks if:

  • The form is signed.
  • The executor has sufficient grade.

Exercise 04: Intern

The final part adds an Intern class capable of creating any of the above forms dynamically:

AForm *Intern::makeForm(std::string formName, std::string target);

The intern identifies the requested form type by name and returns the correct object, or prints an error if unknown.

Final Notes

  • Master exception safety: Always catch by reference.
  • Understand class hierarchies: Base class = abstract behavior, Derived class = specific action.
  • Respect the Orthodox Canonical Form for all classes.

By the end, you’ll know how to build robust, modular, and safe C++ programs that handle complex object relations and error management.
